# 리눅스 기본 명령어 및 셸 변수

## 셸 변수 정리

### 환경 변수

* **`echo $HOME`** : 현재 사용자 홈 디렉터리
* **`echo $USER`** : 현재 로그인한 사용자 이름
* **`echo $SHELL`** : 현재 사용 중인 셸
* ⭐ **`echo $PATH`** : 명령어를 찾는 경로 목록

  * 명령어 실행 파일을 찾기 위해 리눅스가 검색하는 폴더 목록
  * 예시:

    ```bash
    /usr/local/bin:/usr/bin:/bin:/home/user/.local/bin
    ```

    예를 들어 `git` 명령어를 입력했을 때:

    ```text
    /usr/local/bin/git 있나?
    /usr/bin/git 있나?
    /bin/git 있나?
    /home/user/.local/bin/git 있나?
    ```

    찾으면 실행하고, 없으면:

    ```bash
    git: command not found
    ```

* **`echo $PWD`** : 현재 위치 (`pwd` 명령어와 동일)
* **`echo $HOSTNAME`** : 현재 컴퓨터의 호스트명
* **`echo $LANG`** : 시스템 언어 설정
* **`echo $EDITOR`** : 기본 텍스트 편집기 (`nano`, `vim` 등)

---

### 특수 변수

* ⭐ **`echo $?`** : 바로 직전 명령어의 성공/실패 여부

  * `0` = 성공
  * `1 이상` = 실패

* **`echo $$`** : 현재 실행 중인 셸의 PID(프로세스 번호)

* **`echo $!`** : 가장 최근에 백그라운드로 실행한 프로세스 PID

---

### ⭐ 스크립트 전용 변수

스크립트 파일 안에서 사용한다.

`test.sh`라는 스크립트를 만들어서 실행한다고 가정:

```bash
bash test.sh hello world 123
```

* **`echo $0`** : 스크립트 자신의 이름
* **`echo $1`** : 첫 번째 인자
* **`echo $2`** : 두 번째 인자
* **`echo $3`** : 세 번째 인자
* **`echo $#`** : 인자의 개수
* **`echo $@`** : 전달된 인자 전체

---

### 전체 셸 변수 총 정리

| 변수          | 출력 예시               | 의미           |
| ----------- | ------------------- | ------------ |
| `$HOME`     | `/home/mirim`       | 홈 디렉터리 경로    |
| `$USER`     | `mirim`             | 현재 사용자 이름    |
| `$SHELL`    | `/bin/bash`         | 현재 셸         |
| `$PATH`     | `/usr/bin:/bin:...` | 명령어 탐색 경로    |
| `$PWD`      | `/home/mirim`       | 현재 디렉터리      |
| `$HOSTNAME` | `ubuntu-server`     | 호스트명         |
| `$?`        | `0` 또는 `1`          | 직전 명령어 성공 여부 |
| `$$`        | `1234`              | 현재 셸 PID     |
| `$0`        | `script.sh`         | 스크립트 이름      |
| `$1 ~ $9`   | `hello`             | 스크립트 인자      |
| `$#`        | `3`                 | 인자 개수        |
| `$@`        | `hello world`       | 인자 전체        |


> 💡 ⭐ **큰따옴표(`" "`)** 안에서는 `$변수`가 값으로 치환된다.
> ⭐ **작은따옴표(`' '`)** 안에서는 `$변수`가 그대로 출력된다.

```bash
echo "$USER"   # student
echo '$USER'   # $USER
```

## 리눅스 기본 명령어

### 기본 출력 및 시스템 정보

* ⭐ **`echo`** : 화면에 텍스트나 변수 값을 출력
* **`whoami`** : 현재 로그인된 사용자 이름 출력
* ⭐ **`date`** : 현재 날짜 및 시간 출력

```bash
date
date +"%Y-%m-%d"
date +"%H:%M:%S"
```

출력 예시:

```text
Thu Mar 26 09:30:00 KST 2026
2026-03-26
09:30:00
```

* **`clear`** : 터미널 화면을 지움 (`Ctrl + L`과 동일)
* ⭐ **`history`** : 이전에 입력한 명령어 목록 출력

```bash
history
history 10
!42
```

```text
1  ls
2  pwd
3  cd /home
...
```

---

### ⭐ 도움말 보기

* **`man [명령어]`** : 명령어의 공식 매뉴얼 페이지 출력 (`q`로 종료)

```bash
man ls
man cp
man date
```

> 💡 방향키로 스크롤, `/검색어` 입력으로 검색 가능

* **`[명령어] --help`** : 간단한 사용법과 옵션 목록 출력 (`man`보다 간결)

```bash
ls --help
cp --help
date --help
```

### 경로 탐색

### `cd` : 디렉터리 이동

| 표현   | 의미      |
| ---- | ------- |
| `.`  | 현재 디렉터리 |
| `..` | 상위 디렉터리 |

```bash
cd
cd ~
cd /home/mirim
cd documents
cd ..
cd -
```

설명:

```bash
cd                 # 홈 디렉터리 이동
cd ~               # 홈 디렉터리 이동 (동일)
cd /home/mirim     # 절대 경로 이동
cd documents       # 상대 경로 이동
cd ..              # 상위 디렉터리 이동
cd -               # 이전 디렉터리 이동
```

> 💡 `cd`와 `cd ~`는 완전히 동일하다.

---

### ⭐ `pwd` : 현재 작업 디렉터리 출력

```bash
pwd
```

```text
/home/mirim/documents
```

---

### ⭐ `ls` : 디렉터리 목록 조회

| 옵션   | 설명                                     |
| ---- | -------------------------------------- |
| `-a` | 숨김 파일(`.`으로 시작) 포함 출력                  |
| `-l` | 권한, 소유자, 크기, 날짜 등 상세 정보 출력             |
| `-h` | 파일 크기를 KB, MB, GB 단위로 출력 (`-l`과 함께 사용) |
| `-t` | 마지막 수정 시간 기준 내림차순 정렬                   |
| `-R` | 하위 디렉터리까지 재귀적으로 출력                     |
| `-i` | inode 번호 출력                            |

출력 예시:

```bash
ls -al
```

```text
total 32
drwxr-xr-x 4 mirim mirim 4096 Mar 26 09:00 .
drwxr-xr-x 8 mirim mirim 4096 Mar 25 18:00 ..
-rw-r--r-- 1 mirim mirim 220 Mar 26 09:00 .bash_logout
-rw-r--r-- 1 mirim mirim 3526 Mar 26 09:00 .bashrc
drwxr-xr-x 2 mirim mirim 4096 Mar 26 09:05 documents
```

> 💡 옵션은 자유롭게 조합 가능하다.
> 예: `ls -alht`

## ⭐ 파일 및 디렉터리 관리

### `mkdir` : 디렉터리 생성

```bash
mkdir mydir
mkdir dir1 dir2 dir3
mkdir -p parent/child/grandchild
mkdir -p project/{src,docs,test}
```

| 옵션   | 설명                    |
| ---- | --------------------- |
| `-p` | 상위 디렉터리가 없어도 경로 전체 생성 |

---

### `cp` : 파일/디렉터리 복사

```bash
cp file1.txt file2.txt
cp file1.txt /home/mirim/docs/
cp file1.txt file2.txt /backup/
cp -r mydir/ newdir/
cp -a mydir/ backup/
cp -f file1.txt file2.txt
```

| 옵션   | 설명                          |
| ---- | --------------------------- |
| `-a` | 파일 속성(권한, 타임스탬프, 심볼릭 링크) 유지 |
| `-f` | 대상 파일이 있어도 강제 덮어쓰기          |
| `-r` | 디렉터리와 하위 내용 재귀 복사           |

---

### `mv` : 파일/디렉터리 이동 및 이름 변경

```bash
mv oldname.txt newname.txt
mv file.txt /home/mirim/docs/
mv dir1/ /tmp/
mv file1.txt file2.txt /backup/
```

> 💡 `mv`는 이동과 이름 변경이 모두 가능하다.
>
> * 같은 디렉터리 내 이름만 변경 → **이름 변경**
> * 다른 디렉터리로 이동 → **이동**

예시:

```bash
mv report_draft.txt report_final.txt
mv report_final.txt /home/mirim/docs/
```

---

### `rmdir` : 빈 디렉터리 삭제

```bash
rmdir emptydir
rmdir dir1 dir2 dir3
rmdir -p parent/child/grandchild
```

| 옵션   | 설명                    |
| ---- | --------------------- |
| `-p` | 상위 디렉터리도 비어 있으면 함께 삭제 |

> ⚠️ 디렉터리 안에 파일이 있으면 삭제되지 않는다.
> → `rm -r` 사용

---

### ⭐ `rm` : 파일/디렉터리 삭제

```bash
rm file.txt
rm file1.txt file2.txt file3.txt
rm -f file.txt
rm -r mydir/
rm -rf mydir/
```

| 옵션   | 설명                 |
| ---- | ------------------ |
| `-f` | 확인 없이 강제 삭제        |
| `-r` | 디렉터리와 하위 내용을 재귀 삭제 |

> 🚨 **`rm -rf` 주의사항**
>
> * 삭제 시 휴지통 없이 즉시 영구 삭제
> * `rm -rf /` 또는 `rm -rf /*` 실행 시 시스템 전체 삭제
> * 복구가 매우 어려움
> * 실행 전 경로를 반드시 두 번 확인할 것

절대 실행하면 안 되는 예시:

```bash
rm -rf /
rm -rf ~/*
```
