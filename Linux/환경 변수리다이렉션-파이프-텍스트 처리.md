## 쉘이란?
사용자가 입력한 명령어를 받아서 커널(운영체제 핵심)에 전달해주는 프로그램. 통역사 역할.
## 변수

### 셸 변수 vs 환경 변수

| 구분 | 셸 변수 | 환경 변수 |
| --- | --- | --- |
| 범위 | 현재 셸에서만 사용 가능 | 현재 셸 + 자식 프로세스에서도 사용 가능 |
| 생성 방법 | `이름=값` | `export 이름=값` |
| 확인 명령어 | `echo $이름` | `env` 또는 `printenv` |
| 소멸 시점 | 셸 종료 시 | 셸 종료 시 (영구 저장하려면 `.bashrc`) |

---

### ⭐ 셸 변수 만들기

```bash
# 셸 변수 만들기 (= 앞뒤에 공백 없어야 함!)
name="mirim"
age=18
school="마이스터고"

# 변수 값 확인 ($ 붙여야 함!)
echo $name
echo $age
echo "이름: $name, 나이: $age"
```

```
출력 예시:
mirim
18
이름: mirim, 나이: 18
```

>💡 = 앞뒤에 공백이 있으면 에러! <br>
> 변수 값을 읽을 때는 반드시 $를 붙인다. echo name → 글자 name 출력, echo $name → 변수 값 출력

### ⭐ alias - 명령어 별칭 만들기

자주 쓰는 긴 명령어를 짧게 줄일 수 있다.

```bash
# alias 만들기
alias ll="ls -la"
alias myip="ip addr show | grep inet"
alias cls="clear"

# 사용해보기
ll                     # ls -la와 동일하게 동작!
cls                    # clear와 동일

# 현재 등록된 alias 확인
alias
```

> 💡 터미널을 닫으면 alias도 사라진다! 영구 저장하려면 `.bashrc`에 적어야 한다
> 

---

### ⭐ .bashrc에 설정 추가하기

```bash
# vi로 .bashrc 열기
vi ~/.bashrc

# 파일 맨 아래에 다음 내용 추가 (G로 맨 아래 이동 → o로 새 줄)

# === 내가 추가한 설정 ===
alias ll="ls -la"
alias cls="clear"
alias ports="ss -tlnp"
export MY_NAME="mirim"

# ESC → :wq로 저장

# 변경사항 적용 (터미널 재시작 없이 바로 적용)
source ~/.bashrc

# 확인
echo $MY_NAME
ll
```

> 💡 `source ~/.bashrc`를 빼먹으면 변경사항이 적용되지 않는다! <br>
터미널을 닫았다 열거나, `source` 명령어를 실행해야 한다 

## ⭐ 리다이렉션

### 입출력 스트림 개념

리눅스에서는 모든 입력과 출력을 **번호**로 구분한다.

| 번호 | 이름 | 기본 연결 | 의미 |
| --- | --- | --- | --- |
| 0 | 표준 입력 (stdin) | 키보드 | 명령어에 들어가는 입력 |
| 1 | 표준 출력 (stdout) | 화면 (터미널) | 명령어의 정상 결과 |
| 2 | 표준 에러 (stderr) | 화면 (터미널) | 에러 메시지 |

### 리다이렉션 정리표
| 기호 | 의미 | 예시 |
| --- | --- | --- |
| `>` | 표준 출력을 파일에 저장 (덮어쓰기) | `ls > list.txt` |
| `>>` | 표준 출력을 파일에 추가 (이어쓰기) | `echo "추가" >> list.txt` |
| `2>` | 표준 에러를 파일에 저장 | `find / -name x 2> err.txt` |
| `2>&1` | 표준 에러를 표준 출력으로 합치기 | `cmd > all.txt 2>&1` |
| `<` | 파일 내용을 표준 입력으로 전달 | `sort < data.txt` |
| `/dev/null` | 출력을 버림 (쓰레기통) | `cmd 2> /dev/null` |

## ⭐ 파이프 (`|`)

앞 명령어의 출력을 **뒷 명령어의 입력**으로 넘겨준다.

### 파이프 기본 개념

```bash
# 파이프 없이 → 두 단계로 해야 함
ls /etc > temp.txt      # 1단계: 파일에 저장
head -5 temp.txt        # 2단계: 앞 5줄 보기

# 파이프 사용 → 한 줄로 끝!
ls /etc | head -5       # ls 결과를 head에 바로 전달
```

```
출력 예시:
adduser.conf
alternatives
apparmor
apparmor.d
apt
```

> 💡 파이프를 쓰면 중간 파일 없이 명령어를 연결

### ⭐ `tee` - 화면 출력 + 파일 저장 동시에

데이터를 **두 방향**으로 나눈다     

```bash
# 화면에 출력하면서 동시에 파일에도 저장
ls /etc | tee etc_list.txt | head -5

# 이어쓰기 모드 (-a = append)
date | tee -a log.txt
```

### ⭐ `grep` - 패턴 검색 

파일에서 특정 단어(패턴)가 들어간 **줄만 골라내는** 명령어

```bash
# 기본: 파일에서 "ERROR" 포함된 줄 찾기
grep "ERROR" access.log

# 대소문자 구분 없이 찾기 (-i)
grep -i "error" access.log

# 줄 번호와 함께 표시 (-n)
grep -n "ERROR" access.log

# 해당 패턴이 없는 줄만 찾기 (-v) = 반전
grep -v "INFO" access.log

# 검색 결과 개수만 세기 (-c)
grep -c "ERROR" access.log
```

### `sort` - 정렬⭐

텍스트를 줄 단위로 정렬
| 옵션 | 의미 | 예시 |
| --- | --- | --- |
| `-r` | 역순 정렬 (**r**everse) | `sort -r file.txt` |
| `-n` | 숫자로 정렬 (**n**umeric) | `sort -n numbers.txt` |
| `-k N` | N번째 필드 기준 정렬 (**k**ey) | `sort -k2 data.txt` |
| `-t'구분자'` | 필드 구분자 지정 | `sort -t',' -k2 data.csv` |
| `-u` | 중복 제거 후 정렬 (**u**nique) | `sort -u names.txt` |
