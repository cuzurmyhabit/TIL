# 기본 설정과 저장소

## 최초 설정 (한 번만)

```bash
git config --global user.name "홍길동"
git config --global user.email "hong@example.com"
```

커밋마다 이 정보가 기록된다. GitHub 계정 이메일과 맞추는 것을 권장한다.

### 자주 쓰는 추가 설정

```bash
# 기본 브랜치 이름을 main으로
git config --global init.defaultBranch main

# pull 시 merge 대신 rebase (팀 규칙에 따라)
git config --global pull.rebase true

# 에디터 설정 (VS Code 예시)
git config --global core.editor "code --wait"
```

### 설정 확인

```bash
git config --list
git config user.name
```

## 저장소 시작하기

### 기존 프로젝트 클론 (가장 흔함)

```bash
git clone https://github.com/org/frontend-app.git
cd frontend-app
```

SSH를 쓰면 매번 비밀번호 입력이 필요 없다.

```bash
git clone git@github.com:org/frontend-app.git
```

### 새 프로젝트 초기화

```bash
mkdir my-app && cd my-app
git init
```

GitHub에서 빈 저장소를 만든 뒤 연결:

```bash
git remote add origin git@github.com:username/my-app.git
git push -u origin main
```

## 원격 저장소 (Remote)

```bash
# 원격 목록
git remote -v

# 원격 추가
git remote add origin <URL>

# 원격 URL 변경
git remote set-url origin <새_URL>
```

| 이름 | 의미 |
| --- | --- |
| `origin` | 기본 원격 (clone한 저장소) |
| `upstream` | 포크한 원본 저장소 (오픈소스 기여 시) |

## 상태 확인

```bash
git status        # 변경 파일 목록
git log           # 커밋 히스토리
git log --oneline --graph  # 그래프로 보기
git branch        # 브랜치 목록
```

### ⭐ git status 해석

```text
On branch feature/header
Changes not staged for commit:    ← 수정했지만 add 안 함
Changes to be committed:         ← add 했고 commit 대기 중
Untracked files:                 ← Git이 아직 추적 안 하는 새 파일
```

## .gitignore 기초

Git이 **추적하지 않을 파일**을 지정한다. 프론트엔드에서 특히 중요하다.

```gitignore
# 의존성
node_modules/

# 빌드 결과
dist/
build/
.next/
out/

# 환경 변수 (비밀키!)
.env
.env.local

# OS / 에디터
.DS_Store
.vscode/
```

프로젝트 시작 직후 `.gitignore`를 설정하는 습관을 들이자.  
자세한 내용은 [08-프론트엔드-실무-팁.md](./08-프론트엔드-실무-팁.md)에서 다룬다.

## SSH 키 등록 (GitHub)

```bash
# 키 생성
ssh-keygen -t ed25519 -C "hong@example.com"

# 공개키 복사 (macOS)
pbcopy < ~/.ssh/id_ed25519.pub
```

GitHub → Settings → SSH and GPG keys → New SSH key에 붙여넣기.

```bash
# 연결 테스트
ssh -T git@github.com
```

## 정리

| 명령어 | 용도 |
| --- | --- |
| `git config` | 사용자 정보·환경 설정 |
| `git clone` | 원격 저장소 복제 |
| `git init` | 새 저장소 초기화 |
| `git remote add` | 원격 연결 |
| `git status` | 현재 상태 확인 |
