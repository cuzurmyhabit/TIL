# 이벤트 처리

React에서 이벤트는 **합성 이벤트(Synthetic Event)** 로 처리한다.  
브라우저마다 다른 동작을 React가 감싸서 **일관된 API**를 제공한다.

---

## 기본 문법

HTML과 비슷하지만 **camelCase**를 쓴다.

```jsx
function Button() {
  const handleClick = () => {
    alert('클릭!');
  };

  return <button onClick={handleClick}>클릭</button>;
}
```

| HTML | React |
| --- | --- |
| `onclick="..."` | `onClick={handler}` |
| `onchange` | `onChange` |
| `onsubmit` | `onSubmit` |

### ⭐ 함수 호출이 아니라 함수 참조를 넘긴다

```jsx
// ✅ 클릭 시 실행
<button onClick={handleClick}>

// ❌ 렌더 시점에 바로 실행됨
<button onClick={handleClick()}>

// ✅ 인자가 필요할 때
<button onClick={() => handleClick(id)}>
```

---

## 이벤트 객체 (SyntheticEvent)

핸들러는 **이벤트 객체**를 첫 번째 인자로 받는다.

```jsx
function Input() {
  const handleChange = (e) => {
    console.log(e.target.value);
  };

  return <input onChange={handleChange} />;
}
```

### 자주 쓰는 속성

| 속성 | 설명 |
| --- | --- |
| `e.target` | 이벤트가 발생한 DOM 요소 |
| `e.target.value` | input 값 |
| `e.preventDefault()` | 기본 동작 막기 (폼 제출 등) |
| `e.stopPropagation()` | 이벤트 버블링 중단 |

## 폼 이벤트

### input / textarea

```jsx
function SignupForm() {
  const [email, setEmail] = useState('');

  return (
    <input
      type="email"
      value={email}
      onChange={(e) => setEmail(e.target.value)}
    />
  );
}
```

### checkbox

```jsx
const [agreed, setAgreed] = useState(false);

<input
  type="checkbox"
  checked={agreed}
  onChange={(e) => setAgreed(e.target.checked)}
/>
```

### form submit

```jsx
function LoginForm() {
  const [password, setPassword] = useState('');

  const handleSubmit = (e) => {
    e.preventDefault(); // 페이지 새로고침 방지
    // 로그인 로직
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="password"
        value={password}
        onChange={(e) => setPassword(e.target.value)}
      />
      <button type="submit">로그인</button>
    </form>
  );
}
```

## 이벤트와 State → 리렌더 흐름

```text
사용자 클릭
    ↓
onClick 핸들러 실행
    ↓
setState 호출
    ↓
리렌더 스케줄링
    ↓
새 UI 반영
```

```jsx
function LikeButton() {
  const [liked, setLiked] = useState(false);

  return (
    <button onClick={() => setLiked(!liked)}>
      {liked ? '❤️' : '🤍'}
    </button>
  );
}
```

## 이벤트 버블링

자식에서 발생한 이벤트가 부모로 전파된다.

```jsx
function App() {
  return (
    <div onClick={() => console.log('부모')}>
      <button onClick={(e) => {
        e.stopPropagation();
        console.log('자식');
      }}>
        클릭
      </button>
    </div>
  );
}
```

## 핸들러를 Props로 전달

자식이 부모의 State를 바꿀 때 자주 쓰는 패턴이다.

```jsx
function Parent() {
  const [count, setCount] = useState(0);

  return <Counter value={count} onIncrement={() => setCount(count + 1)} />;
}

function Counter({ value, onIncrement }) {
  return (
    <div>
      <span>{value}</span>
      <button onClick={onIncrement}>+1</button>
    </div>
  );
}
```

이름 규칙: `on` + 동사 (`onClick`, `onChange`, `onSubmit`, `onDelete`)

## 키보드 이벤트

```jsx
function SearchInput() {
  const handleKeyDown = (e) => {
    if (e.key === 'Enter') {
      // 검색 실행
    }
  };

  return <input onKeyDown={handleKeyDown} />;
}
```

## 이벤트 핸들러와 성능

인라인 화살표 함수는 **렌더마다 새 함수**가 만들어진다.

```jsx
// 매 렌더마다 새 함수 — 대부분의 경우 문제 없음
<button onClick={() => doSomething(id)}>

// 자식이 React.memo일 때는 useCallback 고려
const handleClick = useCallback(() => doSomething(id), [id]);
```

과도한 최적화는 피하고, **실제 성능 이슈가 있을 때** `useCallback`을 검토한다.

## 정리

| 항목 | 내용 |
| --- | --- |
| camelCase | `onClick`, `onChange` |
| 핸들러 전달 | `onClick={fn}` (호출 결과 X) |
| `e.preventDefault()` | 폼 기본 동작 방지 |
| 제어 컴포넌트 | `value` + `onChange`로 State 동기화 |
| 콜백 Props | 자식 → 부모 상태 변경 |
