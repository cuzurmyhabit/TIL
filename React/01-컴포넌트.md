# 컴포넌트

## 컴포넌트란?

컴포넌트는 **UI를 독립적이고 재사용 가능한 단위로 쪼갠 것**이다.

- 하나의 컴포넌트 = 화면의 한 조각 (버튼, 카드, 헤더 등)
- Props를 받아서 UI를 반환(return)한다
- 같은 컴포넌트를 여러 번, 다른 Props로 재사용할 수 있다

```jsx
function Greeting({ name }) {
  return <h1>안녕하세요, {name}님!</h1>;
}

// 사용
<Greeting name="수민" />
<Greeting name="지수" />
```

## 함수 컴포넌트 vs 클래스 컴포넌트

### ⭐ 함수 컴포넌트 (현재 표준)

```jsx
function Button({ label, onClick }) {
  return <button onClick={onClick}>{label}</button>;
}
```

- Hooks와 함께 쓰는 **권장 방식**
- 코드가 짧고 읽기 쉽다

### 클래스 컴포넌트 (레거시)

```jsx
class Button extends React.Component {
  render() {
    return <button onClick={this.props.onClick}>{this.props.label}</button>;
  }
}
```

- `this`, `render()`, 생명주기 메서드가 필요했던 과거 방식
- 신규 코드에서는 함수 컴포넌트 + Hooks를 사용한다

## 컴포넌트 작성 규칙

### 1. 이름은 반드시 대문자로 시작

```jsx
// ✅ 컴포넌트
function UserCard() { ... }

// ❌ 일반 HTML 태그로 인식됨
function userCard() { ... }
```

React는 **대문자로 시작하는 식별자**만 컴포넌트로 취급한다.

### 2. 하나의 루트 요소를 반환 (또는 Fragment)

```jsx
// Fragment로 여러 요소를 감쌀 수 있음
function Layout() {
  return (
    <>
      <header>헤더</header>
      <main>본문</main>
    </>
  );
}
```

### 3. 컴포넌트는 순수 함수에 가깝게

같은 Props가 들어오면 **항상 같은 UI**를 반환하는 것이 이상적이다.

```jsx
// ✅ 순수 — Props만으로 결과 결정
function Price({ amount }) {
  return <span>{amount.toLocaleString()}원</span>;
}

// ⚠️ 외부 변수를 직접 변경하면 예측이 어려워짐
let count = 0;
function BadCounter() {
  count++; // 렌더마다 외부 상태 변경 — 피할 것
  return <span>{count}</span>;
}
```

## 컴포넌트 합성 (Composition)

큰 컴포넌트 안에 작은 컴포넌트를 **조립**해서 UI를 만든다.

```jsx
function App() {
  return (
    <PageLayout>
      <Header />
      <ArticleList articles={articles} />
      <Footer />
    </PageLayout>
  );
}
```

### children Props

태그 사이에 넣은 내용이 `children`으로 전달된다.

```jsx
function Card({ title, children }) {
  return (
    <div className="card">
      <h2>{title}</h2>
      <div>{children}</div>
    </div>
  );
}

// 사용
<Card title="공지">
  <p>내용입니다.</p>
</Card>
```

## 컴포넌트 분리 기준

| 기준 | 예시 |
| --- | --- |
| **재사용** | Button, Input, Modal |
| **역할 분리** | Header / Sidebar / Content |
| **복잡도** | 한 파일이 200줄 넘으면 분리 검토 |
| **상태 소유** | 상태를 쓰는 단위로 경계 설정 |

```text
❌ 한 파일에 모든 것
App.jsx (500줄)

✅ 역할별 분리
App.jsx
components/
  Header.jsx
  PostList.jsx
  PostItem.jsx
```

## 정리

| 항목 | 내용 |
| --- | --- |
| 컴포넌트 | UI의 재사용 단위 |
| 함수 컴포넌트 | 현재 표준 작성 방식 |
| 합성 | 작은 컴포넌트를 조립해 큰 UI 구성 |
| children | 컴포넌트 태그 사이 내용을 전달하는 Props |
